nt\+Hash = nt\+Hash is a recursive hash function for hashing all possible k-\/mers in a DNA/\+RNA sequence.\hypertarget{md__r_e_a_d_m_e_autotoc_md0}{}\doxysection{Installation \& Usage}\label{md__r_e_a_d_m_e_autotoc_md0}
\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysubsection{btllib (\+Recommended)}\label{md__r_e_a_d_m_e_autotoc_md1}
nt\+Hash is included in \href{https://github.com/bcgsc/btllib}{\texttt{ btllib}}, which can be installed using Conda\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{conda install -\/c bioconda btllib}

\end{DoxyCode}


This installs other classes such as Bloom filters that can be used with nt\+Hash. Also, nt\+Hash can be imported in Python projects via btllib\textquotesingle{}s wrappers. nt\+Hash\textquotesingle{}s code is automatically synced to btllib on each release. Refer to \href{https://github.com/bcgsc/btllib/blob/master/README.md}{\texttt{ btllib\textquotesingle{}s readme}} for usage instructions.\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{Manual Compilation}\label{md__r_e_a_d_m_e_autotoc_md2}
To use nt\+Hash without installing btllib, download the repo and generate a cmake buildsystem in an arbitrary directory (e.\+g. {\ttfamily release}), by running the following command in the project\textquotesingle{}s root\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake -\/S . -\/B release}

\end{DoxyCode}


Then, build the project and its dependencies using\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake -\/-\/build release -\/-\/target all}

\end{DoxyCode}


The project\textquotesingle{}s executable ({\ttfamily ./nthash}) and static library ({\ttfamily libnthash.\+a}) will be generated in the {\ttfamily release} folder.\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsubsection{Executable}\label{md__r_e_a_d_m_e_autotoc_md3}
After building the project, use the output binary file to generate hash values from input data and store the results on disk\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Usage: nthash [options] files }
\DoxyCodeLine{}
\DoxyCodeLine{Positional arguments:}
\DoxyCodeLine{files           Input sequence files [required]}
\DoxyCodeLine{}
\DoxyCodeLine{Optional arguments:}
\DoxyCodeLine{-\/v -\/-\/version    prints version information and exits [default: false]}
\DoxyCodeLine{-\/k              k-\/mer size [required]}
\DoxyCodeLine{-\/o              Output file (for -\/f collect) or directory path [required]}
\DoxyCodeLine{-\/f              Output file organization (create files containing hashes for each 'file', 'record', or 'collect' all hashes into a single file [file] [default: "{}file"{}]}
\DoxyCodeLine{-\/h              Number of hashes per k-\/mer [1] [default: 1]}
\DoxyCodeLine{-\/s              Input spaced seed patterns separated by commas (e.g. 1110111,11011011). Performs k-\/mer hashing if no value provided.}
\DoxyCodeLine{-\/-\/long          Optimize file reader for long sequences (>5kbp) [default: false]}
\DoxyCodeLine{-\/-\/binary        Output hashes in binary files (otherwise plain text) [default: false]}
\DoxyCodeLine{-\/-\/verbose       Print progress to stdout [default: false]}

\end{DoxyCode}


For example, given two input files {\ttfamily 1.\+fa} and {\ttfamily 2.\+fa}, two hash values for each 64-\/mer can be saved to a binary file called {\ttfamily out.\+bin} by running this command in {\ttfamily release}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{./nthash -\/k 64 -\/h 2 -\/o out.bin -\/f collect -\/-\/binary -\/-\/verbose 1.fa 2.fa}

\end{DoxyCode}
\hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysubsubsection{Static Library Usage}\label{md__r_e_a_d_m_e_autotoc_md4}
To use nt\+Hash in a C++ project\+:
\begin{DoxyItemize}
\item Link the code with {\ttfamily nthashlib.\+a} (i.\+e. pass {\ttfamily -\/L path/to/nthash/release -\/l nthashlib.\+a} to the compiler).
\item Add the {\ttfamily include} directory (pass {\ttfamily -\/I path/to/nthash/include} to the compiler).
\item Import nt\+Hash in the code using {\ttfamily \#include \texorpdfstring{$<$}{<}nthash/nthash.\+hpp\texorpdfstring{$>$}{>}}.
\item Access nt\+Hash classes from the {\ttfamily nthash} namespace.
\end{DoxyItemize}
